# 小红书收藏夹智能聚类 - 设计说明文档

## 一、核心功能

### 1. 数据导入功能

#### 具体功能
- **CSV 文件上传**：支持从"设置-账号-个人信息"下载的小红书数据文件
- **JSON 数据导入**：支持直接粘贴 JSON 格式的收藏数据
- **手动文本输入**：支持逐行输入标题内容

#### 设计理由/依据
- **CSV 上传**：小红书官方提供的数据导出格式，用户已有数据无需手动转换
- **JSON 支持**：为开发者和技术用户提供灵活的数据导入方式
- **文本输入**：为测试和快速使用提供最简单的输入方式

**用户体验考虑**：三种导入方式覆盖不同用户场景，降低使用门槛

---

### 2. 向量化与聚类功能

#### 具体功能
- **BGE-small-zh 模型**：本地运行的中文向量化模型
- **Coze Embedding**：云端备选方案
- **自动聚类**：基于轮廓系数自动选择最优 K 值
- **手动聚类**：用户可指定聚类数量（2-20）
- **先降维后聚类**：使用 UMAP 风格布局 + K-Means++

#### 设计理由/依据
- **双模型方案**：
  - BGE-small-zh：中文语义理解准确度高，本地运行保护隐私
  - Coze Embedding：云端服务快速稳定，作为备选方案
- **自动聚类**：普通用户无需理解聚类算法，自动选择最佳结果
- **先降维后聚类**：确保可视化位置与聚类类别一致，避免"标签分离"问题

**技术依据**：
- BGE-small-zh 在中文语义相似度任务上表现优异
- 轮廓系数 (Silhouette Coefficient) 是评估聚类质量的经典指标
- UMAP 保留局部和全局结构，适合可视化

### 2.1 智能聚类命名

#### 具体功能
- **AI 自动命名**：聚类完成后自动生成简短有意义的中文名称
- **语义分析**：基于聚类内容分析主题，生成 2-4 字名称
- **兜底机制**：命名失败时使用默认名称（"聚类 1"）

#### 设计理由/依据
- **为什么要自动命名？**：
  - "聚类 1"、"聚类 2" 缺乏语义信息，用户无法快速理解内容
  - 有意义的名称（如"家居美学"、"AI 工具"）帮助用户快速识别主题
  - 提升产品体验，减少认知负担
- **为什么用 AI 生成？**：
  - 聚类内容多样，人工预设模板无法覆盖所有场景
  - AI 能够理解语义并提取主题关键词
  - 小红书风格的名称更符合平台调性
- **为什么限制在 2-4 字？**：
  - 简洁易记，符合标签命名惯例
  - 过长名称影响界面布局
  - 足够表达核心主题

**技术实现**：
- 使用 LLM (`doubao-seed-1-8-251228`) 生成名称
- 每次取聚类中最多 10 条内容作为样本
- 提示词要求生成小红书风格的简短名称
- 失败时降级为默认名称，不影响主流程

### 3. 知识图谱可视化

#### 具体功能
- **力导向布局**：D3.js 实现，节点自动排列
- **聚类分组**：同类节点聚集在一起
- **节点交互**：
  - 拖拽调整位置
  - 滚轮缩放画布
  - 点击筛选聚类
  - 悬停显示详情
- **节点连接**：同类节点相互连接，聚类间弱连接
- **光晕动效**：聚类背景光晕，增强视觉层次

#### 设计理由/依据
- **力导向布局**：
  - 符合"知识图谱"的认知模型
  - 自动平衡节点分布，无需手动调整
  - 物理模拟（排斥力、引力）直观易懂
- **聚类分组**：
  - 让用户快速识别内容分类
  - 符合"物以类聚"的认知习惯
- **交互设计**：
  - 拖拽：满足用户"整理"的心理需求
  - 缩放：应对大量节点时的浏览需求
  - 筛选：聚焦感兴趣的内容
  - 悬停：提供上下文信息，不打断浏览流程
- **视觉动效**：
  - 光晕：增强聚类边界感
  - 脉冲动画：引导用户注意力

**认知科学依据**：
- 力导向布局模拟物理世界，符合直觉
- 颜色编码利用人类对色彩的快速识别能力
- 即时反馈（悬停、点击）增强用户掌控感

### 4. 数据洞察总览

#### 具体功能
- **统计卡片**：
  - 总收藏数
  - 聚类数量
  - 平均置信度（轮廓系数）
  - 向量化模型
- **聚类分布柱状图**：可视化各聚类内容数量
- **聚类摘要预览**：显示每个聚类的标题和描述
- **收藏列表**：按聚类筛选查看详细内容

#### 设计理由/依据
- **统计卡片**：
  - 提供全局视图，快速了解数据概貌
  - 数字量化让用户产生"掌控感"
- **柱状图**：
  - 比较不同聚类的大小差异
  - 视觉化比文字更易理解
- **摘要预览**：
  - 让用户快速浏览聚类主题
  - 无需进入详情即可判断聚类质量
- **详细列表**：
  - 满足用户"查看原始数据"的需求
  - 验证聚类结果的准确性

**信息设计原则**：
- 层次化信息：概览 → 详情
- 渐进式披露：先看摘要，再决定是否深入
- 可视化优先：图表优于表格

### 5. AI 跨界联想

#### 具体功能
- **双聚类选择**：选择两个不同聚类进行联想
- **拖放式交互**：直观的选择方式
- **AI 创意生成**：基于两个聚类的跨界联想
- **结果展示**：以小红书语境生成创意方案

#### 设计理由/依据
- **跨界联想的价值**：
  - 发现隐藏关联，激发创作灵感
  - 打破思维定势，创新内容方向
  - 符合小红书"种草"和"发现"的核心价值
- **双聚类选择**：
  - 两个对象足够简单，易于理解
  - 过多的选择会增加认知负担
  - 对比关系更清晰
- **AI 创意生成**：
  - 利用 LLM 的关联推理能力
  - 小红书语境确保输出符合平台调性
  - 温度参数 (0.9) 确保创意性

**心理学依据**：
- 组合创新理论：新创意 = A + B
- 远程联想理论：越不相关的组合越有创意潜力
- 减少认知负荷：每次只处理两个对象

### 5.1 帖子联想

#### 具体功能
- **Shift + 点击选择**：在知识图谱中快速选择两个帖子
- **可视化选择**：选中的节点变大并有白色边框高亮
- **帖子卡片展示**：顶部显示已选中的两个帖子
- **一键联想**：点击按钮生成创意灵感
- **流式输出**：实时显示 AI 生成的创意

#### 设计理由/依据
- **Shift + 点击交互**：
  - 不干扰原有的聚类筛选功能
  - 符合用户心智模型（Windows/文档多选）
  - 无需额外 UI 元素，界面更简洁
- **节点高亮**：
  - 视觉反馈明确
  - 白色边框区分于普通节点
  - 尺寸增大便于识别
- **创意对比粒度更细**：
  - 聚类联想：主题级别的创意
  - 帖子联想：内容级别的创意
  - 用户可选择不同粒度

**交互设计依据**：
- 直接操作：点击比拖放更自然
- 即时反馈：节点变化提供视觉确认
- 流式输出：打字机效果增强期待感

### 6. 智能问答

#### 具体功能
- **自然语言提问**：用户可以用自然语言提问
- **语义检索**：基于向量相似度找到相关内容
- **引用回答**：AI 基于相关内容生成回答，并标注引用来源
- **参考来源展示**：列出回答引用的收藏内容

#### 设计理由/依据
- **问答的价值**：
  - 从"被动浏览"到"主动探索"
  - 快速找到需要的信息
  - 深度挖掘内容价值
- **RAG（检索增强生成）**：
  - 避免 LLM 幻觉，确保回答基于真实内容
  - 引用机制增加可信度
  - 向量检索确保语义匹配
- **Top-K 检索**：
  - 返回 5 条最相关的内容，平衡广度和精度
  - 过多内容会增加上下文长度，降低响应质量

**认知科学依据**：
- 问答是人类获取信息最自然的方式
- 引用机制符合"可追溯性"原则
- 渐进式展示降低认知负担

## 二、界面设计

### 1. 整体风格

#### 设计特点
- **深色主题**：背景色 `#0f0f12`
- **红色强调色**：主色调 `#ef4444`
- **圆角卡片**：统一使用 `rounded-3xl`
- **半透明玻璃态**：`backdrop-blur-md` + `bg-white/5`

#### 设计理由/依据
- **深色主题**：
  - 科技感和现代感
  - 减少眼睛疲劳（长时间使用场景）
  - 聚焦内容，减少视觉干扰
  - 符合"知识图谱"的科技产品定位
- **红色强调色**：
  - 与小红书品牌色呼应
  - 情感化设计：红色代表活力、创意
  - 对比度强，引导用户注意力
- **圆角卡片**：
  - 现代设计趋势（2024+）
  - 柔和视觉感受，减少攻击性
  - 增加触摸友好性
- **玻璃态**：
  - 层次感：背景模糊凸显前景内容
  - 轻盈感：半透明减少视觉重量
  - 呼应"知识图谱"的透明、互联概念

**设计心理学**：
- 色彩理论：红色激发情感和行动
- 格式塔原则：圆角创造和谐感
- 视觉层次：透明度创造深度

**交互设计原则**：
- 渐进式披露：信息按需显示
- 状态反馈：清晰表达可用性

### 4. 信息层次设计

#### 具体功能
- **三层信息架构**：
  1. 顶层：统计卡片（全局概览）
  2. 中层：可视化图表（趋势和关系）
  3. 底层：详细列表（原始数据）

#### 设计理由/依据
- **三层信息架构**：
  - 符合"从宏观到微观"的认知流程
  - 用户可按需深入，无需一次性接收所有信息
  - 满足不同使用场景（快速浏览 vs 深度分析）
- **颜色编码**：
  - 快速识别聚类，无需阅读文字
  - 视觉记忆强，易于关联
  - 跨视图一致性（图谱、总览、联想）
- **字体层级**：
  - 明确信息重要性
  - 引导视线流动
  - 符合排版设计原则

**信息设计理论**：
- 渐进式披露：信息分层呈现
- 视觉层次：大小、颜色、位置表达重要性
- 一致性原则：跨界面保持相同编码

## 四、技术选型

### 1. Next.js 16 + React 19

#### 选择理由
- **SSR/SSG 支持**：首屏渲染速度快，SEO 友好
- **App Router**：现代化的路由方案，支持 Server Components
- **React 19 新特性**：更好的性能和开发体验
- **TypeScript**：类型安全，减少运行时错误

### 2. D3.js

#### 选择理由
- **强大的可视化能力**：支持复杂的力导向布局
- **灵活性**：可自定义任何视觉效果
- **生态成熟**：文档完善，社区活跃
- **性能优秀**：处理上千节点无压力

### 3. shadcn/ui

#### 选择理由
- **现代化设计**：符合当前设计趋势
- **高度可定制**：基于 Radix UI 和 Tailwind
- **轻量级**：按需引入，无额外负担
- **TypeScript 原生**：与项目技术栈一致

### 4. BGE-small-zh

#### 选择理由
- **中文优化**：在中文语义理解上表现优异
- **轻量级**：模型较小，加载速度快
- **本地运行**：保护用户隐私，无需网络
- **开源免费**：无成本

### 5. LLM SDK (Coze)

#### 选择理由
- **流式输出**：更好的用户体验
- **多模型支持**：8 个模型可选
- **简单易用**：API 设计简洁
- **生产级**：支持错误处理和重试

## 五、设计决策总结

### 1. 用户中心设计
- 三种导入方式降低使用门槛
- 自动聚类减少用户决策负担
- 直观的交互降低学习成本

### 2. 视觉一致性
- 统一的深色主题
- 一致的圆角和玻璃态风格
